A diferencia de sistemas monolíticos Docker se enfoca a una arquitectura microservicios. Esta tecnológica utiliza contenedores para la construcción y despliegue de aplicaciones de forma estandarizada. Haciendo referencia a la introducción, los contenedores permiten el empaquetar aplicaciones y sus dependencias en una imagen virtual. Como analogía, las maquinas virtuales son creadas para almacenar aplicaciones, y sus dependencias se rigen al sistema operativo,lo que requiere un uso total de recursos para una aplicación que corre en un determinado SO; Docker permite crear imágenes y usar exclusivamente las dependencias de la aplicación y no exige la instalación de todo un SO. Esto optimiza recursos en el sistema.
